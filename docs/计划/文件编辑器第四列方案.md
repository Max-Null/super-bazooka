# 文件编辑器第四列 + MD 选中发送到聊天

> 状态：设计中 | 2026-07-06

## 目标

1. **第四列文件预览/编辑面板**：FilePreviewModal 从全屏弹窗改造为右侧面板，对话在左、文件在右，同屏操作
2. **MD 选中 Tip**：在 Markdown 编辑 Tab 中选中文本后，弹出输入框填写修改建议，点击发送直接发给当前会话

## 核心思路

**不新建组件，直接改造 FilePreviewModal**。它已有：

- CodeMirror 6 编辑器 + 语言检测 + Ctrl+S 保存 + dirty 跟踪
- 编辑/预览 Tab 切换
- HTML 预览中的 DOM 选择器 → 发送到聊天（已有 `sendDomToChat`）
- MD → docx 转换
- 文件类型检测（text/html/markdown/docx/image）

改造方案：给 FilePreviewModal 加 `mode` prop，`"modal"`（默认，全屏弹窗）和 `"panel"`（内联面板，无遮罩、无 Teleport）。

## 目标布局

```
┌──────────┬──────────────────┬────────────┬──────────────────────┐
│ 会话 Rail │   ChatPanel      │ FilePanel  │ FilePreviewModal     │
│ (40px)   │   (router-view)  │ (可折叠)    │ mode="panel"         │
│ + 侧栏   │                  │ 文件树+预览 │ 编辑/预览 + 选中发送  │
└──────────┴──────────────────┴────────────┴──────────────────────┘
```

用户在中间区域：左边对话、右边编辑/选择，不用切弹窗。

## 设计决策

### 1. FilePreviewModal 改造

**新增 prop：**
```ts
mode?: "modal" | "panel"  // 默认 "modal" 保持向后兼容
```

**mode="modal"（现有行为，不变）：**
- `<Teleport to="body">` + 全屏遮罩
- ChatPanel 中预览附件、FilePanel 中放大预览 继续使用

**mode="panel"（新增）：**
- 不用 Teleport，直接内联渲染
- 无遮罩、无 backdrop
- 左侧拖拽把手调节宽度（复用 FilePanel drag-to-resize 模式）
- 关闭按钮 → emit close，AppShell 隐藏该列
- 面板宽度默认 ~400px，可拖拽 280-700px

**模板差异：**
```
modal:  Teleport > .fixed.inset-0.z-50 > 居中卡片
panel:  直接渲染 > .h-full.flex.flex-col > 内容区
```

### 2. AppShell 布局

第四列插在 FilePanel 右侧：

```vue
<!-- AppShell.vue 模板 -->
<div class="flex-1 flex overflow-hidden">
  <!-- 会话 Rail -->
  ...
  <!-- ChatPanel -->
  <main class="flex-1 flex overflow-hidden">
    <router-view />
  </main>
  <!-- FilePanel -->
  <FilePanel ... />
  <!-- 第四列：文件预览/编辑面板 -->
  <FilePreviewModal
    v-if="panelFile"
    mode="panel"
    :file="panelFile"
    @close="panelFile = null"
  />
</div>
```

**打开方式：**
- FileTree 双击文件 → `emit('openInEditor', entry)` → FilePanel 转发 → AppShell 设置 `panelFile`
- FileTree 单击文件 → 现有行为不变（FilePanel 内联预览）
- 目录双击 → 现有行为不变（导航进入）

**状态管理：**
```ts
// AppShell.vue
const panelFile = ref<{ name: string; path: string } | null>(null);
provide("openFileInPanel", (f: { name: string; path: string }) => { panelFile.value = f; });
```

FileTree 通过 inject 获取 `openFileInPanel`，双击文件时调用。

### 3. MD 选中 Tip

**触发条件：**
- `fileKind === "markdown"` 且 `activeTab === "edit"`
- CodeMirror 选区非空

**实现：**
- 在 `EditorView.updateListener` 中检测选区变化
- 选区非空 → 在选区末尾下方显示 Tip
- 选区为空 → 隐藏 Tip

**定位：**
- `view.coordsAtPos(selection.main.head)` 获取选区末尾屏幕坐标
- Tip 用 `position: fixed`，z-50 以上
- 上下边界检测：超出视口底部则翻到选区上方

**Tip UI：**
```
┌─────────────────────────────────┐
│ 📝 "选中的文本前40字..."        │
│                                 │
│ ┌─────────────────────────────┐ │
│ │ 输入修改建议...              │ │
│ └─────────────────────────────┘ │
│                         [发送]  │
└─────────────────────────────────┘
```

**发送逻辑（复用 sendDomToChat 模式）：**
- 构造消息：
  ```
  请修改以下 Markdown 内容：

  ```markdown
  {选中的完整文本}
  ```

  修改建议：{用户输入}
  ```
- 通过 `emitChatCommand` 发给 ChatPanel → 自动填入并发送
- 发送后关闭 Tip，清除 CodeMirror 选区

### 4. 不做的

- **不做多 Tab**：panel 模式单文件，切换文件就换内容
- **不做语法检查/LSP**
- **不做文件树右键"在面板中打开"菜单项**：双击即可
- **不在 FilePreview（只读内联预览）中加选中 Tip**

## 实施步骤

### Step 1: FilePreviewModal 加 panel 模式
- 添加 `mode` prop，默认 `"modal"`
- panel 模式下：不用 Teleport、不用遮罩、内联 flex 布局
- 模板用 `v-if` 分支区分 modal/panel 渲染结构
- 验证：modal 模式行为不变（ChatPanel 附件预览、FilePanel 放大预览）

### Step 2: AppShell 第四列
- flex 布局增加第四列槽位
- `panelFile` 状态 + `provide("openFileInPanel")`
- 左侧拖拽把手（复用 FilePanel drag 逻辑）
- FileTree 双击文件 → inject openFileInPanel

### Step 3: MD 选中 Tip
- CodeMirror updateListener 监听选区
- Tip 浮层定位 + textarea + 发送按钮
- emitChatCommand 发送到聊天
- 边界检测 + 翻转到上方

### Step 4: i18n + 测试
- 中英文案（编辑器面板、Tip 输入框 placeholder、发送按钮）
- FilePreviewModal panel 模式测试
- MD 选中 Tip 交互测试

## 风险 & 边界

- **modal 模式向后兼容**：ChatPanel 附件预览、FilePanel 放大预览继续用 modal 模式，不受影响
- **列数增加导致宽度紧张**：FilePanel 和 panel 列默认折叠/隐藏，按需打开
- **双击冲突**：目录双击 = 导航，文件双击 = 打开面板，文件单击 = 预览——无冲突
- **panel 关闭时未保存**：关闭前检查 dirty，弹 confirm（复用现有 confirmClose）
