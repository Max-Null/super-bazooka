# 设计决策参考

> 从 CLAUDE.md 移出的完整设计决策列表。写代码需要理解"为什么这样做"时查阅。
> CLAUDE.md 仅保留最核心的 5 条用于 compact 后存活。

## 进程与协议

1. **三线程进程模型**: 参考 TOKENICODE 架构，Waiter 管理进程生命周期 + Stdout Reader 解析流 + Stderr 转发错误
2. **NDJSON 双工协议**: CC 子进程以 `--output-format stream-json --input-format stream-json --permission-prompt-tool stdio` 启动（不带 `--print`，两者互斥）。用户消息通过 stdin 写入 NDJSON `{"type":"user","message":{...}}`。初始化后发 `set_permission_mode` 控制请求告知 CC 权限模式
3. **control_request/response 审批流**: CC 通过 stdout 发 `control_request(can_use_tool/hook_callback)` → `protocol.rs` 提取 `request_id`、`tool_name`、`tool_input` → 前端 `ChatPanel.vue` 渲染审批栏（通用工具）或问答弹窗（`tool_name === "AskUserQuestion"`）→ 用户操作后构造 `control_response` 写入 stdin。`hook_callback` 不自动回复，走统一的用户审批流程
4. **interrupt 优雅停止**: 用户点击停止→先发 `control_request(subtype:"interrupt")` 到 stdin，等 3 秒让 CC 优雅退出，超时才 `pm.kill()`
5. **AskUserQuestion 问答弹窗**: 识别 `tool_name === "AskUserQuestion"` 时不显示通用审批栏，弹出 `ModalShell` 内 radio/checkbox 选项组 + Other 自由输入。答案格式 `updatedInput:{questions,answers}`，answers key 精确匹配 question 原文

## 数据与持久化

6. **SQLite 持久化**: rusqlite bundled 模式，WAL journal，4 表 schema（sessions, messages, settings, approved_scenarios）+ 3 个索引
7. **用户消息单写**: 用户消息只由 Rust 后端 `send_message` 保存，前端不再重复保存，避免历史回显双份
8. **流事件去重**: `useStreamProcessor` 对 `stream-event` 按 sessionId + content 前缀去重，防止 CLI 重复输出
9. **会话管理**: 前端 ourId (UUID v4) ↔ 后端 claude_session_id (CLI 返回)，映射关系存储在 `sessions.cli_session_id`

## Provider 与 API

10. **不传 --model 给 CLI**: Claude CLI 使用自己的 `~/.claude/settings.json` 配置模型；GUI 的 model 设置存储在 session 记录中供参考
11. **权限同步**: `sync_permission_settings()` 在每次 spawn 前将 auto/default 模式写入 `~/.claude/settings.json`
12. **Provider 配置持久化**: 每次切换 provider 时，apiKey/baseUrl/model 自动保存到 SQLite `settings` 表（key=`provider_config:{id}`）。切换顺序：`saveCurrentConfig()` → `restoreConfig(id)` → `providerId = id`（必须先恢复值再切 providerId，防止 watcher 中间态错误写入 settings.json）
13. **OpenAI 兼容端点 URL 规范化**: 所有调 `/v1/chat/completions` 的地方必须先 `trim_end_matches("/anthropic").trim_end_matches("/v1")` 再拼接 URL。Rust 端统一用 `fn openai_base(base_url)` 处理
14. **翻译和描述生成模型**: 翻译、MCP 描述生成等轻量 API 调用使用 `CLAUDE_CODE_SUBAGENT_MODEL` 环境变量（fallback `deepseek-chat`）。Rust 端统一用 `fn subagent_model()` 读取

## UI

15. **亮色主题 CSS 顺序**: `[data-theme="light"]` 必须在 `:root` 之后，禁止 `@import` 独立亮色 CSS 文件（会被提升到 `:root` 前导致变量被覆盖）
16. **ModalShell 三段式布局**: header（`#header` 插槽 + 关闭按钮）/ body（default slot，`overflow-y-auto`）/ footer（`#footer` 插槽，`v-if="$slots.footer"`）。子组件用插槽填充各段，不要自己在 default slot 里造 flex 三段式
17. **字号设置**: 支持小/中/大（14/18/22px），通过 `data-font-size` 属性 + `html { font-size }` CSS 实现全局 rem 缩放。ModalShell 宽度用 rem 适配。ManagePanel 内 `text-[*px]` 全部转为 `text-[*rem]` 以跟随字号
18. **拼音搜索**: 命令面板支持拼音首字母搜索，`lib/pinyin.ts` 覆盖 3755 个常用汉字
19. **工具名 i18n**: `tools.Bash`→`命令行`、`tools.Write`→`写入文件` 等，`zh.json`/`en.json` 双份。`ChatPanel.vue` 和 `MessageBubble.vue` 共用 `toolLabel()` 映射
20. **思考计时分段**: 每段 tool_use 前独立计时（`thinkingDurationMs`），总耗时 = 各段求和。去掉了实时 `setInterval` 计时器，不受审批暂停影响

## 管理面板

21. **ManagePanel 8合1管理**: 统一管理 8 个 Tab（plugins/mcp/skills/agents/hooks/memory/permissions/styles），通过 Tab 切换时清空数据并重新加载
22. **MCP 配置多源扫描**: 扫描 `~/.claude.json` → `~/.claude/.mcp.json` → 项目根 `.mcp.json` → 已启用插件的 `.mcp.json` → 运行时连接状态。注意：`~/.claude/settings.json` 的 `mcpServers` 字段会被 CLI 静默忽略
23. **Skills/Agents 管理多源扫描**: 同时扫描自定义目录和已启用插件的缓存目录，按来源分组显示。Skills 点击直接执行（`/skill-name`）而非打开编辑
24. **Agent 使用状态追踪**: chat store 维护 `usedAgents: Set<string>`，`addToolUse()` 检测 Agent/Task 工具调用并提取 `subagent_type`。ManagePanel 的 Agent Tab 根据 Set 标记绿点/灰点，用过的排前面
25. **Hook 管理 schema 适配**: ManagePanel 的 Hooks Tab 适配 CC 新版 Hook schema（`事件 → [matcher → {hooks: [{type, command}]}]`），不再假设扁平结构

## 其他

26. **文件写入安全域**: Rust `write_file` 命令只允许写入 `~/.claude/` 子树，拒绝越界路径
27. **可扩展命令注册**: `useCommandRegistry` 允许各组件动态注册命令面板命令，`CommandPalette.vue` 启动时收集所有注册命令
28. **Mock 优先测试**: `tauri-mock.ts` 提供完整 mock，Playwright 测试在浏览器中运行无需 Tauri
29. **SessionStart Hook 行为规则注入**: 全局行为规则（Ponytail、Karpathy、桥梁容错）由 PowerShell 脚本在 SessionStart 时注入，无 "may not be relevant" 免责。CLAUDE.md 保留骨架版用于 `/compact` 后存活
30. **Ponytail 精简模式 GUI**: 工具栏新增 Ponytail 下拉（同权限/深度样式），选中直接发送 `/ponytail <mode>`。Setting 面板同位置设默认值。状态持久化到 localStorage
31. **DeepSeek 兼容**: 前端设置面板配置 API Key/Base URL/Model，连接测试走 DeepSeek `/v1/chat/completions`

## 版本号管理

项目版本号以 `package.json` 为准，通过 Vite `define` 注入 `__APP_VERSION__` 到 UI（设置页 about 显示）。

发版时需同步更新三处：

| 文件 | 字段 | 说明 |
|------|------|------|
| `package.json` | `version` | **源头** — Vite 构建时注入 UI |
| `src-tauri/tauri.conf.json` | `version` | 安装包版本号（MSI/NSIS） |
| `src-tauri/Cargo.toml` | `version` | Rust 二进制版本号 |

```bash
grep '"version"' package.json src-tauri/tauri.conf.json src-tauri/Cargo.toml
```
