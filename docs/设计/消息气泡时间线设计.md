# 消息气泡时间线 设计方案

> 2026-07-16 | 已实施 | MessageBubble.vue + useStreamProcessor.ts + chat.ts

## 一、问题

Claude Code 的每条 assistant 回复由**多步骤交替**组成：思考 → 工具调用 → 执行结果 → 再思考 → 最终文本。传统的"先展示所有思考，再展示所有工具，最后文本"的设计完全打乱了原始时序，用户无法理解 Claude 的实际推理链条。

## 二、方案

引入 `ContentBlock` 顺序数组，严格保持 CC 输出的原始块顺序。在每条 assistant 消息气泡内以**左侧竖线 + 圆点**的垂直时间线进行渲染，每个块一个节点，按类型着色。

### 数据模型

```typescript
// chat.ts
interface ContentBlock {
  type: "text" | "thinking" | "tool_use" | "tool_result";
  content?: string;        // text/thinking 块的文本
  toolUse?: ToolUse;        // tool_use 块的工具信息
  toolResult?: ToolResult;  // tool_result 块的执行结果
}

interface Message {
  // ...
  contentBlocks?: ContentBlock[]; // 时间线数组，保持 CC 原始顺序
}
```

### 视觉模型

```
┌─ sb-timeline ────────────────────────────────────┐
│                                                    │
│  ●─ thinking  💭 思考过程（默认折叠）               │
│  │  └ 🧠1.2s  "分析用户需求，确定需要读取文件..."     │
│  │                                                 │
│  ●─ tool_use  🔧 Read(文件名.ts)                   │
│  │  └ ⚡0.3s { file_path: "..." }                 │
│  │  └ Input:  { file_path: "src/lib.ts" }         │
│  │  └ Output: 文件内容…                            │
│  │                                                 │
│  ●─ thinking  💭 思考过程                          │
│  │  └ 🧠0.8s  "根据文件内容，需要修改..."           │
│  │                                                 │
│  ●─ tool_use  🔧 Edit(文件名.ts)                   │
│  │  └ ⚡0.2s { file_path, old_string, new_string } │
│  │  └ Input:  { ... }                             │
│  │  └ Output: (空)  ✓                              │
│  │                                                 │
│  ●─ text  💬 Markdown 最终回复                     │
│                                                    │
│  — 思考中 —      ← 状态行（流式中波动动画）          │
│  2.5s · 1.2K tok · $0.0015                        │
└────────────────────────────────────────────────────┘
```

### 圆点配色

| 块类型 | 边框色 | 图标 | CSS 类 |
|--------|--------|------|--------|
| `thinking` | `var(--amber)` 琥珀 | 💭 | `.tl-thinking .sb-timeline-dot` |
| `tool_use` | `var(--violet)` 紫罗兰 | 🔧 | `.tl-tool_use .sb-timeline-dot` |
| `tool_result` | `var(--accent)` 主色 | — | 不独立渲染，嵌套在 tool_use 内部 |
| `text` | `var(--border-bright)` | 💬 | `.tl-text .sb-timeline-dot` |

## 三、数据流

```
CC stdout NDJSON
  ├─ stream_event (text_delta, content_block_start, …)
  └─ assistant (message.content[] = content_blocks)
       │
       ▼
  useStreamProcessor.buildContentBlocks(raw, existing)
       │
       ▼
  chat.setContentBlocks(blocks)
       │
       ▼
  MessageBubble.vue  template v-for="block in message.contentBlocks"
       │
       ▼
  sb-timeline-block  × N  （thinking / tool_use / text）
```

### 合并策略：`buildContentBlocks`

流式事件和完整 `assistant` 事件交替到达，需要在已有数组上智能合并：

| 块类型 | 合并规则 | 原因 |
|--------|---------|------|
| `thinking` / `text` | 反向查找同类上一个块；新内容以旧内容开头 → **替换**；否则 **追加** | CC 全量事件携带累积内容（`startsWith` 去重）；DeepSeek 增量事件携带新 token（无法 startsWith，直接拼接） |
| `tool_use` | 按 `id` 去重 | 同一个工具调用出现在多次 assistant 事件中 |
| `tool_result` | 按 `toolUseId` 去重 | — |

### 旧格式兼容：`synthesizeBlocks`

历史消息或非 `content_blocks` 协议的消息，从旧字段重建时间线：

```
thinking 有内容 → { type: "thinking" }
每个 toolUses 条目 → { type: "tool_use" } + { type: "tool_result" }（如有结果）
text 有内容 → { type: "text" }
```

### 持久化与恢复

- **保存**：`result` 事件时将完整 `contentBlocks` 写入 JSON blob → SQLite `messages.content`
- **恢复**：`loadMessages()` 时从 JSON 解析 `contentBlocks`；旧存档无此字段则调用 `synthesizeBlocks()` 重建

## 四、计时系统

### 分阶段计时

`useStreamProcessor` 维护三个时间戳：

```
thinkingStart → 思考阶段开始
toolExecStart → 工具执行阶段开始
lastToolUse   → 当前正在执行的工具引用
```

事件驱动的切换：

```
thinking 事件到达
  ├─ toolExecStart > 0 → 结算上一个工具的执行耗时 → 归零
  └─ 设置 thinkingStart = Date.now()

tool_use 事件到达
  ├─ thinkingStart > 0 → 结算该工具的思考耗时 → 归零（popThinkingDuration）
  └─ 设置 toolExecStart = Date.now()

result 事件到达
  ├─ 结算最后的 toolExecStart（没有后续 tool_use 触发 pop）
  └─ 结算最后的 thinkingStart（附加到最后一个工具的 thinkingDurationMs）
```

### 计时同步到 UI

`syncBlockTimings()` 在每次 `assistant` 事件后执行，将 `toolUses` 数组中的计时信息（`thinkingDurationMs`、`executionDurationMs`）同步到 `contentBlocks` 对应 tool_use 条目上。渲染时直接读取 block 上的计时字段：

- `thinkingDurationMs`：显示在下一个 tool_use 块上 → `🧠1.2s`
- `executionDurationMs`：显示在该 tool_use 块上 → `⚡0.3s`，超 5 秒标红 `tl-slow`
- 流式中无 `executionDurationMs` 时：根据 `startedAt` 计算实时流逝 → `⚡1.5s` + `tl-live` 动画

## 五、块交互行为

### thinking 块

- 默认**折叠**，只显示摘要行（`💭 思考过程` + 前 60 字 + 计时）
- 流式中最后一个 thinking 块显示绿色闪烁圆点 + 波浪文字动画 `— 思 考 中 —`
- 点击 `<summary>` 展开完整思考文本

### tool_use 块

- 默认**折叠**，摘要行显示（工具名 + 执行时间 + 输入摘要 + 结果状态标记）
- 展开后分三个区域：Input（输入的 JSON）、Output（执行结果，绿色）或 Error（红色）
- tool_result **不单独渲染**，嵌套在对应 tool_use 块的 Output 区内
- 无结果时（等待中）只显示 Input，流式计时持续跳动

### text 块

- 直接渲染，**不折叠**
- 内容经 `MarkdownRenderer` 渲染（GFM 完整支持）
- 是时间线的最后一个块（通常）

### 状态行

时间线底部常驻一行状态文字：

| 状态 | 显示 | 效果 |
|------|------|------|
| 流式中 | `— 思 考 中 —` | 逐字波浪动画（每个字 0.15s 相位差） |
| 已停止 | `— 已停止 —` | 珊瑚色（`var(--coral)`） |
| 已完成 | `— 已完成 —` | 灰色 |

## 六、CSS 线段绘制

每个 `.sb-timeline-block` 的 `::after` 伪元素绘制连接线：

```css
.sb-timeline-block::after {
  content: '';
  position: absolute;
  left: 10px;        /* 圆点中心水平位置 */
  top: 12px;         /* 当前圆点中心 (top 4 + 圆点 16/2 = 12) */
  bottom: -20px;     /* 延伸到下一块圆点 (margin 8 + top 4 + 圆点 16/2 = 20) */
  width: 2px;
  background: var(--border-dim);
}
/* 末块不画线 */
.sb-timeline-block.tl-last::after { display: none; }
```

每块 `padding-left: 26px` 为圆点 + 连线留出空间。整体 `position: relative` 保证 `::after` 绝对定位基准。

## 七、与聊天级时间线的关系

| 维度 | 消息气泡时间线 (`sb-timeline`) | 聊天导航 (`ChatTimelineNav`) |
|------|-------------------------------|------------------------------|
| **粒度** | 一条 Claude 回复的**内部步骤** | 整个会话的**用户消息** |
| **数据源** | `message.contentBlocks` | `messages.filter(role==='user')` |
| **方向** | 气泡内左侧垂直排列 | 聊天区右侧垂直排列 |
| **用途** | 展示思考→工具→回复完整链路 | 快速跳转到任意用户消息 |
| **交互** | 折叠/展开、计时、流式实时 | 点击跳转、hover 预览 |
| **压缩** | 无（步数通常 < 20） | 有（消息数可能 100+） |

## 八、边界情况

| 场景 | 行为 |
|------|------|
| 纯文本回复（无思考/工具） | 仅一条 text 块，无线段 |
| 多轮工具调用（5+ tools） | 全部按顺序渲染，每条独立节点 |
| 工具无结果（超时/中断前） | tool_use 块不显示 Output 区；流式计时持续跳动 |
| 历史消息无 contentBlocks | `synthesizeBlocks()` 从 `thinking`/`toolUses`/`content` 重建 |
| DeepSeek 后端（增量事件） | `buildContentBlocks` 判断是否 `startsWith` 决定替换/追加 |
| 流式中最后一个 thinking | 显示绿色脉冲圆点 + "思考中" 波浪动画，不显示摘要文字 |
| 空文本回复（仅工具执行） | 底部显示 `{n} 个工具已执行` 兜底 |
| 用户消息 | 不走时间线，直接渲染纯文本气泡 |
