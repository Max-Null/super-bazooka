## 颜色语义

| 变量 | 用途 |
|------|------|
| `--accent` | 主色调——主按钮背景、激活态文字 |
| `--accent-dim` | 主色调暗色——边框、分隔 |
| `--accent-glow` | 主色调发光——选中背景、状态消息、badge 强调 |
| `--amber` | 琥珀色——思考块边框/图标 |
| `--amber-glow` | 琥珀色发光——思考块背景 |
| `--coral` | 珊瑚色——拒绝按钮、错误态 |
| `--bg-root` | 最深背景——主按钮文字 |
| `--bg-surface` | 表面背景——弹窗面板 |
| `--bg-elevated` | 抬升背景——badge、hover |
| `--bg-hover` | 悬停背景——ghost button hover |
| `--border-dim` | 暗边框——分隔线、badge 边框 |
| `--border-default` | 默认边框——弹窗面板外框 |
| `--text-muted` | 弱文字——关闭按钮、辅助信息 |
| `--text-secondary` | 次要文字——ghost 按钮、描述 |
| `--text-primary` | 主要文字——正文 |
| `--text-bright` | 亮文字——标题 |

## 基础控件

### 按钮

所有按钮统一使用以下语义类，禁止在组件中手写 `px-* py-* rounded-*`。

```
主操作（accent 背景，白色文字）
class="btn-primary"

次要操作（透明背景，hover 变色）
class="btn-ghost"

拒绝操作（coral 边框 + 文字）
class="btn-ghost" style="color:var(--coral); border-color:var(--coral)"

导出操作（虚线边框）
class="btn-ghost" style="border-style:dashed"
```

**CSS 定义**（`main.css`）：
```css
.btn-primary {
  padding: 0.5rem 0.75rem; border-radius: 0.375rem;
  font-size: 0.75rem; font-weight: 500;
  background: var(--accent); color: var(--bg-root);
}
.btn-primary:hover { opacity: 0.8; }

.btn-ghost {
  padding: 0.375rem 0.75rem; border-radius: 0.375rem;
  border: 1px solid transparent;
  font-size: 0.75rem; color: var(--text-secondary);
}
.btn-ghost:hover { background: var(--bg-hover); }
```

**按钮选用表**：

| 场景 | 类名 | 示例 |
|------|------|------|
| 提交/确认/批准/执行 | `btn-primary` | 执行计划、提交答案、确认重命名 |
| 跳过/取消/关闭/次要操作 | `btn-ghost` | 跳过、取消、关闭 |
| 拒绝/停止 | `btn-ghost` style coral | 拒绝工具、停止计划 |
| 导出/保存 | `btn-ghost` style dashed | 导出计划 |

### 标签徽章 badge

```
默认（暗色背景 + 细边框）
class="badge"

强调（accent 主题色）
class="badge badge--accent"
```

**CSS**：
```css
.badge {
  display: inline-flex; align-items: center;
  padding: 0.125rem 0.5rem; border-radius: 999px;
  font-size: 0.625rem; font-weight: 500;
  background: var(--bg-elevated);
  border: 1px solid var(--border-dim);
  color: var(--text-secondary);
}
.badge--accent {
  background: var(--accent-glow);
  border-color: transparent;
  color: var(--accent);
}
```

**选用**：
- 键盘快捷键提示 → `badge`
- 问题分类标签（AskUserQuestion header）→ `badge badge--accent`

### 输入框 input-plain

弹窗内统一使用：
```
普通             → class="input-plain"
抬升背景（弹窗用）→ class="input-plain input-plain--elevated"
```

**CSS**：
```css
.input-plain {
  width: 100%; font-size: 0.75rem; padding: 0.5rem 0.75rem;
  border-radius: 0.375rem; border: 1px solid var(--border-dim);
  outline: none; color: var(--text-primary);
  caret-color: var(--accent);
}
.input-plain--elevated {
  background: var(--bg-elevated);
  border-color: var(--border-default);
  color: var(--text-bright);
}
```

## 弹窗外壳 ModalShell

### 架构

```
┌─────────────────────────────────────────┐
│ modal-shell-overlay                      │  ← 全屏半透明遮罩，点击关闭
│  ┌─────────────────────────────────────┐ │
│  │ modal-shell-panel (sm/md/lg/xl)    │ │  ← 白色面板，圆角 + 阴影
│  │  ┌─────────────────────────────────┐│ │
│  │  │ modal-shell-header              ││ │  ← 标题 + 关闭按钮
│  │  ├─────────────────────────────────┤│ │
│  │  │ modal-shell-body (可滚动)       ││ │  ← 内容区
│  │  ├─────────────────────────────────┤│ │
│  │  │ modal-shell-footer (可选)       ││ │  ← 按钮区
│  │  └─────────────────────────────────┘│ │
│  └─────────────────────────────────────┘ │
└─────────────────────────────────────────┘
```

### 组件接口

```typescript
props: {
  open: boolean
  position?: "center" | "top"   // center=居中，top=靠上（问答/计划用）
  size?: "sm" | "md" | "lg" | "xl"
}
slots: {
  header   // 标题区
  default  // 内容区
  footer   // 按钮区（可选）
}
emit: { close: [] }
```

### 宽度预设

| size | 宽度 | 适用弹框 |
|------|------|----------|
| `sm` | 28rem | 关于、重命名 |
| `md` | 34rem | 默认、AskUserQuestion |
| `lg` | 36rem | 上下文用量、导出预览 |
| `xl` | 38rem | 计划审核 |

### 垂直位置

| position | 行为 | 适用弹框 |
|----------|------|----------|
| `center`（默认）| 居中，max-height: 85vh | 关于、导出、重命名、上下文 |
| `top` | 距顶 14vh，max-height: 70vh | AskUserQuestion、计划审核 |

### CSS 定义

见 `main.css` 中 `modal-shell-*` 系列类。所有属性收拢在 CSS 中，组件模板中不再出现 Tailwind token。

### 标题样式

使用 `.modal-shell-title`，禁止在 header 中重复写 `text-sm font-semibold`：

```html
<template #header>
  <span class="modal-shell-title">标题文字</span>
</template>
```

### 关闭行为

- **Escape 键**：组件内部注册 keydown 监听
- **点击遮罩**：`@click.self="emit('close')"`
- **关闭按钮**：header 右侧 × 按钮

## 各弹框规格

### 1. AskUserQuestion（选项询问）

| 属性 | 值 |
|------|-----|
| 触发条件 | `pendingControlRequest.tool_name === 'AskUserQuestion'` |
| position | `top` |
| size | `md` |
| 头部 | 标题文字 |
| 内容 | 问题列表：`badge--accent` 分类 + radio/checkbox + Other 自由输入 |
| 底部按钮 | `btn-ghost` 跳过 + `btn-primary` 提交 |
| 关闭行为 | 跳过 = 直接清空审批队列（`resolveControlRequest`，不发送 control_response 到 CC）|

### 2. ExitPlanMode / 计划审核

| 属性 | 值 |
|------|-----|
| 触发条件 | `isPlanPending()` 或 `showPlanModal` |
| position | `top` |
| size | `xl` |
| 头部 | 标题 + 计划文件路径 |
| 内容 | Markdown 渲染计划文档 + 反馈输入框 |
| 底部按钮（审批中）| `btn-primary` 执行 + `btn-ghost`×3（继续设计/停止/导出） + 自定义反馈按钮 |
| 底部按钮（已保存）| `btn-ghost` 导出 + `btn-ghost` 关闭 |
| 关闭行为 | 驳回 = 发送反馈给 CC；直接关闭 = 仅隐藏 |

### 3. 上下文用量 ContextUsageModal

| 属性 | 值 |
|------|-----|
| 触发条件 | 命令面板 / ContextIndicator 点击 / toolbar |
| position | `center` |
| size | `lg` |
| 内容 | 独立组件 `ContextUsageModal.vue`，Token 用量饼图 + Compact 按钮 |

### 4. 导出预览

| 属性 | 值 |
|------|-----|
| 触发条件 | `export-session` 命令 / 导出按钮 |
| position | `center` |
| size | `lg` |
| 头部 | 文件名 |
| 内容 | MarkdownRenderer → 预览 |
| 底部按钮 | `btn-ghost` 取消 + `btn-primary` 选择目录导出 |

### 5. 重命名会话

| 属性 | 值 |
|------|-----|
| 触发条件 | `rename-session` 命令 / F2 |
| position | `center` |
| size | `sm` |
| 内容 | 单行输入框（autofocus）|
| 底部按钮 | `btn-ghost` 取消 + `btn-primary` 确认 |

### 6. 关于

| 属性 | 值 |
|------|-----|
| 触发条件 | `about` 命令 |
| position | `center` |
| size | `sm` |
| 内容 | 应用名 + 描述 + 版本号 + 技术栈 |

### 7. 管理面板 ManagePanel

| 属性 | 值 |
|------|-----|
| 触发条件 | `manage-*` 命令 / AppShell 头部按钮 |
| size | `xl` |
| 内容 | 独立组件，多标签页管理（插件/MCP/技能/Agent/Hook/记忆/权限/样式）|

### 8. 命令面板 CommandPalette

| 属性 | 值 |
|------|-----|
| 触发条件 | `Ctrl+K` |
| size | 独立定位，非 ModalShell |
| 特殊 | 搜索式交互，支持拼音首字母 |

### 9. 更新日志 ChangelogDialog

| 属性 | 值 |
|------|-----|
| 触发条件 | 版本号变化时自动弹出 |
| 内容 | Markdown 渲染的 CHANGELOG |
| 底部按钮 | `btn-primary` 关闭 |

## 审批条

CC 需要用户确认工具调用时，在 `bottom-notices` 容器内显示审批条。

```
┌──────────────────────────────────────────────┐
│ ▌ 允许 Bash(command)？          [允许] [拒绝] │
└──────────────────────────────────────────────┘
```

| 元素 | 样式 |
|------|------|
| 容器 | `.approval-bar`（bg-elevated 背景 + 圆角卡片 + accent 细竖条）|
| 允许按钮 | `btn-primary` |
| 拒绝按钮 | `btn-ghost` + coral 色 |

**触发排除**：AskUserQuestion 和 ExitPlanMode 不显示审批条——它们有自己的专用弹窗。

**CSS**：
```css
.approval-bar {
  padding: 0.5rem 1rem; display: flex; align-items: center; gap: 0.75rem;
  max-width: 48rem; margin: 0 auto 0.25rem; width: 100%;
  background: var(--bg-elevated);
  border: 1px solid var(--border-dim);
  border-radius: 0.375rem;
}
```

## 工作清单 TodoPanel

CC 通过 TodoWrite / TaskCreate 工具产出的任务列表，显示在 `bottom-notices` 容器内审批条下方。

| 状态 | 样式 |
|------|------|
| `pending` | `bg-root` + 可见边框 + 灰色文字 + 序号 ○ |
| `in_progress` | `accent-glow` + accent 边框 + 脉冲 ● 动画 |
| `completed` | `bg-root` + 透明边框 + 删除线 + 灰色 ✓ |
| `deleted` | 不显示 |

- 任务保持 CC 产出原始顺序，不按状态重新分组
- 每项前有序号（1, 2, 3…）
- 圆角卡片样式，`max-width: 48rem` 居中

**CSS**：见 `TodoPanel.vue` scoped style。

## 浮层与通知

### 状态消息 status-toast

操作反馈（切换工作区、重命名、导出、删除、分叉等），自动 5 秒消失。
定位在会话区顶部居中悬浮，不遮挡底部的审批条、TodoPanel、工具栏。

```
┌─ chat-area ──────────────────────────────────┐
│           ✅ 文件保存成功 — report.md          │  ← status-toast
│  ┌─ 消息列表 ───────────────────────────────┐ │
│  │  …                                      │ │
```

```css
.status-toast {
  position: absolute; top: 0.75rem; left: 50%; transform: translateX(-50%);
  z-index: 30; pointer-events: none;
}
.status-float-enter-active { transition: opacity 0.25s ease-out, top 0.25s ease-out; }
.status-float-leave-active { transition: opacity 0.2s ease-in, top 0.2s ease-in; }
.status-float-enter-from { opacity: 0; top: 0; }
.status-float-leave-to { opacity: 0; top: 0; }

.status-pill {
  font-size: 0.75rem; padding: 0.25rem 0.75rem;
  border-radius: 999px;
  background: var(--accent-glow); color: var(--accent);
}
```

### 调试日志浮层

| 元素 | 描述 |
|------|------|
| Debug 日志 | CC 原始 stream-json 输出 |
| Stderr 日志 | CC 进程标准错误输出 |
| 触发 | InputBar 工具栏按钮 |
| 定位 | 绝对定位在输入栏上方，Teleport to body |

### 滚动到底按钮

消息列表未在底部时，悬浮在输入栏上方的 ↓ 按钮。Transition 动画。

### 吸顶问题横幅

用户消息滚出可视区后，在消息列表顶部显示当前问题摘要。纯 CSS sticky 实现。

## 底部通知区 bottom-notices

审批条和工作清单的共享容器——`shrink-0; position: relative; z-index: 10`，子元素从上到下自然堆叠，统一样式（圆角卡片 + `max-width: 48rem` 居中）。

```html
<div class="bottom-notices">
  <div v-if="…" class="approval-bar">…</div>
  <TodoPanel />
</div>
```

```css
.bottom-notices {
  flex-shrink: 0; position: relative; z-index: 10;
}
```

## 维护规则

1. **新增弹框**：优先复用 ModalShell——指定 `size` 和 `position`，用 `<template #header/footer>` 填内容
2. **按钮**：只用 `btn-primary` / `btn-ghost`，不手写 padding/rounded/font-size
3. **颜色**：只用 CSS 变量，不硬编码色值
4. **类名**：新增语义 class 收在 `main.css`，不在组件 style 块零散定义（组件 scoped 除外）
5. **3+ Tailwind token**：必须提取为语义 class
6. **修改弹框**：对照本文档确保按钮/间距/颜色与同类弹框一致
7. **弹框关闭**：测试或非 CC 场景下，跳过弹框应直接调 `resolveControlRequest` 清队列（不走 `handleDeny`），避免 `sendStdin` 阻塞
