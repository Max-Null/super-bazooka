## 二、FileKind 类型系统

### 当前已支持

| FileKind | 扩展名 | 解析方式 | 默认 Tab | 编辑 | 选取→发送 |
|----------|--------|---------|---------|------|----------|
| `text` | 其他文本文件 | readFileContent → CodeMirror | edit | ✅ | — |
| `html` | .html .htm | readFileContent → iframe blob | preview | ✅ | ✅ DOM 点击 |
| `markdown` | .md .mdx .markdown | readFileContent → MarkdownRenderer | preview | ✅ | ✅ MD 编辑选取 |
| `docx` | .docx .doc | mammoth → HTML | preview | ❌ | ❌ |
| `image` | 常见图片格式 | readFileBase64 → `<img>` | preview | ❌ | — |
| `unsupported` | 二进制格式 | — | — | ❌ | — |

### 本次计划新增

| FileKind | 扩展名 | 解析方式 | 默认 Tab | 编辑 | 选取→发送 |
|----------|--------|---------|---------|------|----------|
| `xlsx` | .xlsx .xls .csv | SheetJS → HTML 表格 | preview | ❌ | ✅ 拖拽选区 |
| `pptx` | .pptx | JSZip → 提取文本 | preview | ❌ | ✅ 文本划选 |

### pptx 实现说明

PPTX 视觉渲染（Canvas / 第三方库）经过多轮迭代均不理想——文字溢出、样式丢失、库对中文 PPTX 兼容性差。
最终采用**纯文字提取方案**：

- JSZip 解包 → 按 `<p:sp>` 形状提取 Y 坐标 + `<a:t>` 文字 → 按 Y 从上到下排列
- 相邻形状 Y 间距 > ~52px 切新段落组，虚线分隔
- 每个 slide 独立展示，从上到下连续排列
- 文字为真实 DOM（`<pre>`），原生可选可复制
- 零新依赖（JSZip 已有）
- 独立组件 `PptxPreview.vue`

## 四、选取→发送至对话（统一设计）

### 4.1 统一交互模式

所有预览 Tab 的选取→发送统一为以下流程：

```
用户操作（点击/拖拽/划选）→ 显示选区高亮 → mouseup 弹出浮动 tip
→ tip 显示选区摘要 → 用户点「发送到对话」→ 选区作为卡片出现在输入框上方附件栏
→ 用户填写修改意见 → 正常发送 → CC 收到文件 + 选区 + 修改意见
```

### 4.2 选区卡片（输入框上方附件栏）

选区不直接发送给 CC。用户点「发送到对话」后，选区以**卡片形式**出现在输入框上方的附件栏，和文件附件并排展示，用图标区分类型：

```
┌──────────────────────────────────────────────────────┐
│ 📋 查看计划   📎 报告.docx   [-] README.md L15-16   │
│                                          [×] [×] [×] │
├──────────────────────────────────────────────────────┤
│ [输入框：请把"12 个项目"改为"14 个项目"……]  [发送]  │
└──────────────────────────────────────────────────────┘
```

**卡片样式**（复用现有 `sb-attachment-bar` 的 chip 样式）：

| 来源 | 图标 | 卡片显示内容 | 示例 |
|------|------|------------|------|
| 右键菜单「添加到对话」 | 📎 | 文件名 | `📎 预算表.xlsx` |
| Excel 拖拽选区 | ⊞ | `文件名 Sheet!范围` | `⊞ 预算表.xlsx Sheet1!B2:D5` |
| DOCX/PPTX 文本选区 | [-] | `文件名` | `[-] 报告.docx` |
| MD 预览文本选区 | [-] | `文件名 L行号`（唯一定位时）或 `文件名` | `[-] README.md L15-16` |
| MD 编辑选区 | [-] | `文件名 L行号` | `[-] README.md L15-18` |
| HTML DOM 选区 | < > | `文件名 <tag>` | `< > index.html <header>` |

- 文件附件一直用 📎，选区用不同图标区分
- 点击卡片上的 × 可移除
- 发送消息时，所有卡片内容按 4.3 格式拼入消息正文前面

### 4.3 CC 接收到的消息格式（按文件类型）

用户点发送后，附件栏所有卡片的内容 + 用户输入拼成最终消息。CC 收到的是**文件路径 + 定位坐标 + 当前内容 + 修改指令**。

#### Excel（拖拽选区）

```
在 `预算表.xlsx` 的 Sheet1!B2:D5 区域，当前内容为：

|   | B     | C     | D     |
|---|-------|-------|-------|
| 2 | 50000 | 48000 | 2000  |
| 3 | 12000 | 13500 | -1500 |
| 4 | 8000  | 8000  | 0     |
| 5 | 15000 | 14000 | 1000  |

请修改为...
```

- 文件路径明确，Sheet 名 + 坐标精确，数据以 Markdown 表格呈现
- CC 能直接定位到 Sheet → 列号 → 行号，不需要再读文件

#### MD 预览（鼠标划选文本）— 利用原始源码定位

**关键**：MD 文件在 `content.value` 中保有原始 Markdown 源码。用户划选后，在源码中搜索选中文本，按匹配次数分级：

```
匹配 1 次（唯一定位）：
在 `README.md` 第 15-16 行选中了以下内容：

克隆项目后运行 `npm install`。

请改为...

匹配 3 次（无法唯一定位）：
在 `README.md` 中选中（此文本出现 3 次，以下为第 1 处，位于「## 安装」下方）：

克隆项目后运行 `npm install`。

⚠️ 请先确认修改位置后再操作。
```

- 唯一匹配 → 等同编辑 Tab 的精确度，带行号
- 多次匹配 → 附带标题上下文 + 出现次数 + 警告提示，CC 需要确认位置

#### DOCX / PPTX（鼠标划选文本）— 结构上下文定位

DOCX/PPTX 是二进制格式，无法映射到源码行号。通过文档结构定位：

```
在 `报告.docx` 的「第三章 预算分析」→「3.2 支出明细」小节中选中：

> 2026 年上半年，科技部门完成了 12 个项目，总预算执行率达到 95%。

请把"12 个项目"改为"14 个项目"。
```

- **结构上下文**：取选区上方最近的标题层级（mammoth 转出的 HTML 保留 `<h1>`~`<h6>`）
- PPTX 同理，取幻灯片编号 + 该页第一个标题
- 足够消歧——同一小节内重复文本极少；若 CC 匹配到多处，会自行向用户确认

#### MD 编辑 Tab（CodeMirror 选区）

```
在 `README.md` 第 15-18 行选中了以下内容：

```
## 安装

克隆项目后运行 `npm install`。
```

请补充系统要求。
```

- 有精确行号——编辑 Tab 用 CodeMirror，选区起点和终点行号已知
- CC 可根据行号直接定位修改，也可以确认行号对应内容后精确替换
- 无需重新读文件（文件内容已在编辑器中，与磁盘一致）

#### HTML 预览（点击 DOM 元素）

```
在 `index.html` 中选中了这个元素：

<header class="main-header" id="top-banner">
  Super Bazooka
</header>

请修改为...
```

- 元素标签 + class + id → CC 可在文件中 grep 定位
- HTML 文本内容附带，帮助 CC 确认匹配到正确位置

> DOCX 用 mammoth 转为 HTML 后 `v-html` 渲染，`getSelection()` 可获取文本。跨元素划选（如 `<h1>` + `<p>` 交界）选区可能截断，需要在 tip 中做 trim 处理。

### 4.4 不统一的例外

- **HTML DOM 选择器**：保持点击元素模式（与文本划选不同，DOM 选取语义不同）
- **MD 编辑 Tab**：保持行号定位模式（编辑场景需要精确行号，不能降级为纯文本）

### 4.5 浮动 Tip（composable 而非独立组件）

三套 tip 各有特定发送逻辑（DOM 调 `sendDomToChat`、MD 编辑带建议输入框、文本划选直接发送），统一为一个组件会把所有差异变成 props/emit，样板代码比复用的还多。参考四列重构中砍掉 `PanelDragHandle` 的经验，改用 composable 管理定位+显隐：

```ts
// useSelectionTip.ts — 统一管理 tip 定位和显隐
function useSelectionTip() {
  const tipVisible = ref(false)
  const tipPos = ref({ left: "0px", top: "0px" })
  const tipSummary = ref("")

  function showTip(summary: string, anchorRect: DOMRect) {
    tipSummary.value = summary.slice(0, 80)
    tipPos.value = {
      left: `${Math.min(anchorRect.left, window.innerWidth - 320)}px`,
      top: `${Math.min(anchorRect.bottom + 6, window.innerHeight - 160)}px`,
    }
    tipVisible.value = true
  }
  function hideTip() { tipVisible.value = false }

  return { tipVisible, tipPos, tipSummary, showTip, hideTip }
}
```

**FilePreviewPanel 内保留三套 tip 模板**（各自用 `<Teleport>` + composable 的数据），不复用同一个组件——各自发送回调不同，强行统一是过度抽象。

### 4.6 内联预览选取（FilePanel）

FilePanel 底部的内联 `<FilePreview>` 也应支持选取→发送。和内联预览中的"在第四列打开"按钮逻辑一致：内联预览 + 第四列 FilePreviewPanel 共用一个选取机制，不区分位置。

实现：FilePanel 通过 inject 获取选取→发送的管道（或在 FilePanel 内部就地处理），不重复建设。

### 4.7 发送数据通道

统一使用 `emitChatCommand` → `useChatCommandBus` 管道。文本划选（DOCX/PPTX/MD 预览）和 MD 编辑选取合并为一个命令，行号可选：

| 命令前缀 | 数据类型 | ChatPanel 处理 |
|---------|---------|---------------|
| `attach-dom:` | HTML 元素信息 | 存入 domSnippet，发送时前置 |
| `selection:` | 文件+文本+行号(可选) | 存入 selectionSnippets，发送时拼入正文 |
| `excel-selection:` | 文件+坐标+Markdown 表格 | 存入 selectionSnippets，发送时拼入正文 |

**`selection:` 格式**：
- 无行号：`在 \`文件.md\` 中选中：\n> 选中文本`
- 有行号：`在 \`文件.md\` L12-15 选中：\n\`\`\`\n选中文本\n\`\`\``

## 五、编辑 Tab

### 5.1 编辑器配置

- **引擎**：CodeMirror 6
- **支持语言**：TS/JS/Python/Rust/CSS/HTML/JSON/MD/SQL/XML/YAML
- **功能**：语法高亮、行号、撤销/重做、Ctrl+S 保存、脏状态指示
- **主题**：oneDark（深色）

### 5.2 编辑生命周期

```
file 变化 → destroyEditor() → 读文件 → createEditor(doc)
tab 切换到 edit → maybeCreateEditor()
tab 切换离开 edit → destroyEditor()
Ctrl+S → saveFile() → saveFileContent(path, text) → dirty = false
关闭面板 → dirty 为 true 时 confirm("未保存")
```

### 5.3 哪些类型可编辑

- ✅ text、html、markdown — 文本内容，CodeMirror 直接编辑
- ❌ docx、xlsx、pptx — 二进制/格式复杂，不提供编辑
- ❌ image — 非文本格式

## 七、数据流汇总

```
路径 A: 文件双击 → 第四列
  FileTree.onDblClick → inject(openFileInPanel) → AppShell.panelFile = f
  → FilePreviewPanel 挂载 → 读文件 → 渲染

路径 B: 右键 → 附件栏
  FileTree.addToChat → CustomEvent("attach-files")
  → ChatPanel.onAttachFiles → attachedFiles.push

路径 C: 预览选取 → 附件栏卡片 → 发送 (统一)
  第四列选取:
    [Excel]     mousedown→mousemove→mouseup → tip → 「发送」→ emitChatCommand("excel-selection:")
    [DOCX/PPTX/MD预览] mouseup → getSelection() → tip → 「发送」→ emitChatCommand("selection:")
    [HTML]      click DOM → tip → 「发送」→ emitChatCommand("attach-dom:")
    [MD 编辑]   CodeMirror select → tip → 「发送」→ emitChatCommand("selection:") (带行号)
  内联预览选取 (FilePanel):
    同上 → emitChatCommand → 同管道
  → ChatPanel.commandWatcher → 选区卡片出现在附件栏（📎 旁）
  → 用户输入修改意见 → handleSend → 卡片内容拼入消息正文前
```

## 八、边界与错误处理

### 8.1 xlsx 解析错误

| 错误场景 | UI 表现 |
|---------|--------|
| 文件损坏 / 非 xlsx 格式 | 预览区显示「文件格式错误或已损坏」 |
| 密码保护 | 「此工作簿受密码保护，无法预览」 |
| 空工作簿（无 Sheet） | 「此工作簿没有数据」 |

### 8.2 pptx 解析错误

| 错误场景 | UI 表现 |
|---------|--------|
| 文件损坏 / 非 ZIP 格式 | 「文件格式错误或已损坏」 |
| .ppt 旧格式 | 「.ppt 旧格式不支持，请转为 .pptx」 |
| 无幻灯片 | 「此演示文稿没有幻灯片」 |
| 纯图片幻灯片 | 「（图片幻灯片，无可提取文本）」 |

### 8.3 大数据量策略

xlsx 超大数据集（>1000 行）时 `sheet_to_html` 生成完整 `<table>` 会卡顿。策略：
- 默认渲染前 1000 行，超出部分截断，底部显示「已截断，仅显示前 1000 行」
- 多 Sheet 时每个 Sheet 独立截断
- 未来可选：分页 / 虚拟滚动

### 8.4 选区边界处理

- DOCX（mammoth→HTML）：跨元素划选可能截断，tip 中 `trim()` 处理首尾空白
- Excel：合并单元格取实际 `<td>` 跨越区域
- 空选区不显示 tip

## 九、设计决策

1. **Excel/PPT 不提供编辑**：格式复杂（合并单元格、公式、动画），做编辑等于造 Office。只做预览+选取发送，让 CC 去改文件。

2. **PPTX 不用专用库**：JSZip + XML 正则提取文本足够，省一个依赖。旧 .ppt 格式不支持（二进制 OLE）。

3. **CSV 复用 xlsx 流程**：先读文本再交给 SheetJS 解析为单 sheet 的 workbook，统一渲染路径。

4. **选取→发送不做实时同步**：用户选取后必须点「发送到对话」，不自动发送。避免误操作。

5. **MD 编辑 Tab 的选取保持行号机制**：编辑场景下 CC 需要知道行号才能精确修改文件，纯文本选取丢掉了这个关键信息。

6. **浮动 tip 用 composable 而非独立组件**：当前三套 tip 各有特定发送回调（DOM 调 `sendDomToChat`、MD 编辑带建议输入框、文本划选直接发送），统一为一个组件 = 样板代码 > 复用收益。用 `useSelectionTip` composable 管理定位+显隐，tip 模板保留在 FilePreviewPanel 内部。
