# 消息时间线分页压缩 设计方案

> 2026-07-16 | 已实施 | ChatTimelineNav.vue | 更新 2026-07-17

## 一、问题

右侧消息时间线导航（`ChatTimelineNav`）将每条用户消息渲染为一个竖排圆点。消息多时（50+），圆点溢出可视区域，无法区分和操作。

## 二、方案

**分页式压缩**：默认仅显示活跃消息附近的圆点，中间用 `…` 省略。按住 `Alt` 临时展开全部。

### 视觉模型

```
20 条消息，活跃在第 10 条（WINDOW=2）：

  ●        ← 第 1 条（首点常驻）
  …        ← "7 条消息（按住 Alt 展开全部）"
  ○        ← 第 8 条
  ○        ← 第 9 条
  ●        ← 第 10 条（活跃，高亮发光）
  ○        ← 第 11 条
  ○        ← 第 12 条
  …        ← "7 条消息（按住 Alt 展开全部）"
  ●        ← 第 20 条（尾点常驻）
```

### 压缩算法

```
total = userMessages.length
active = clamp(activeIndex, 0, total - 1)
WINDOW = 2

visibleSet = {0, total-1} ∪ {active-WINDOW … active+WINDOW}

遍历 0..total-1：
  在 visibleSet 中 → 渲染 Dot
  连续不在 visibleSet → 合并为一个 Ellipsis
```

首尾两点常驻。窗口范围与首尾重叠时自动去重，不产生冗余省略号。

### 交互矩阵

| 操作 | 压缩模式 | 展开模式（按住 `Alt`） |
|------|----------|----------------------|
| 可见圆点 | 活跃窗口 + 首尾 + `…` | 全部 |
| hover 圆点 | 容器级事件：所有可见圆点**同时**弹出 tooltip | 所有圆点同时弹出 tooltip |
| 圆点间移动 | tooltip 稳定显示，不闪烁 | 同左 |
| 点击圆点 | 跳转 + 脉冲动画（dot-pulse 600ms） | 跳转 + 脉冲动画 |
| hover `…` | title 显示"隐藏 N 条 + 快捷键提示" | —（不渲染） |
| 点击 `…` | 跳到隐藏区间的中间位置 | — |
| `Alt` 按下 | 切换到展开模式 | — |
| `Alt` 松开 | 恢复到压缩模式 | 恢复到压缩模式 |
| 窗口失焦 | 恢复压缩（防 Alt+Tab 卡态） | — |

### 快捷键选择

选择 `Alt` 而非 `Shift` 或 `Ctrl`：

| 键 | 排除理由 |
|----|---------|
| `Shift` | 与文本选区扩展冲突 |
| `Ctrl` | 与 `Ctrl+C/V` / `Ctrl+W` 等高频操作冲突 |
| `Alt` | 无冲突，Windows 菜单栏激活在 webview 内无副作用 |

## 三、实现

### 组件：`ChatTimelineNav.vue`

```ts
// 核心类型
interface DotItem      { type: "dot";      index: number }
interface EllipsisItem { type: "ellipsis"; label: string; jumpTo: number }
type TimelineItem = DotItem | EllipsisItem

// 压缩计算（computed）
const timelineItems = computed<TimelineItem[]>(() => {
  const total = userMessages.value.length
  if (showAll.value || total <= WINDOW * 2 + 3) {
    return userMessages.value.map((_, i) => ({ type: "dot", index: i }))
  }
  // 首点 + 前置省略 + 窗口 + 后置省略 + 尾点
  // ...
})
```

### 关键参数

| 参数 | 值 | 说明 |
|------|---|------|
| `WINDOW` | 2 | 活跃点前后各保留数，控制压缩密度 |
| `WINDOW*2+3` | 7 | 低于此阈值不压缩，全部显示 |
| tooltip 截断 | 80 字符 | 消息摘要最大长度 |

### i18n

| Key | 中文 | English |
|-----|------|---------|
| `chat.timelineEllipsis` | `{n} 条消息` | `{n} messages` |
| `chat.timelineExpandHint` | `按住 Alt 展开全部` | `Hold Alt to expand all` |

### CSS 要点

- `.chat-timeline-nav--expanded`：展开时加左侧微弱渐变背景，提示用户处于临时展开状态
- `.chat-timeline-ellipsis`：14px 字号 `…`，`margin-top: -1px` 补偿字体基线偏移，hover 时变 accent 色
- `.chat-timeline-dot`：间距扩大至 `gap: 20px`（圆心距 28px，tooltip 高 ~22px 不重叠）
- `.chat-timeline-dot--just-clicked`：点击脉冲动画 600ms（scale 2→1.2→1.4 + 光晕扩散）
- 容器 `pointer-events` 移除（需要接收 `mouseenter`/`mouseleave`，24px 窄条不影响聊天区）

### tooltip 交互细节

- **容器级事件**：`@mouseenter`/`@mouseleave` 绑在 `.chat-timeline-nav` 容器上，而非每个圆点
- `mouseenter`/`mouseleave` 不冒泡——只在鼠标跨过容器边界时触发一次，圆点间移动无事件
- `showTooltips` 布尔值控制所有圆点的 `<Transition name="tooltip-fade">` 同步显隐

## 四、边界情况

| 场景 | 行为 |
|------|------|
| 消息 ≤ 7 条 | 不压缩，全部显示 |
| 活跃在起点（index 0） | 无前置省略，窗口自然延伸 |
| 活跃在末尾（index N-1） | 无后置省略，窗口自然延伸 |
| `activeIndex = -1`（初始态） | fallback 为末尾（`total - 1`） |
| 按住 Alt 时切窗（Alt+Tab） | `window.blur` 重置 `showAll`，防止卡态 |
| 消息动态增加（流式） | `watch(messages.length)` 触发重算，`nextTick` 后更新 |

## 五、测试

- 全量 277 测试通过，零回归
- 压缩逻辑不改变 `scrollTo` emit 语义，父组件（ChatPanel）无感知
- 键盘事件在两处 `onUnmounted` 中分别清理，无泄漏

## 六、待观察

- `WINDOW=2` 适合 1080p 高度。若 4K 屏或超长会话，可视情况调至 3-4
- 展开模式下的全量 tooltip 在极端会话（200+ 消息）下可能有渲染压力，届时可加虚拟化
